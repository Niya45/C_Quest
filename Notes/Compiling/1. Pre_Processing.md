
## 1. Pre-Processing : the step before processing starts

- A macro processor that is aText-substitution tool to handles directives starting with # symbol
- 1st stage in compiling : .c files to .i files (expanded source code)
- the processor doesn't understand c syntax; but operates strictly on text and tokens


- **Phase 1** : Line Splicing
    - Preprocessor locates backslash-newline sequences and removes them. (allows a single logical line of code to be spread across multiple physical line)
- **Phase 2** : Tokenization and Comment removal
    - Source code is broken in to processing tokens, comments are replaced by a single space character
- **Phase 3** : Header Inclusion (#include)
    - The preprocessor literally copies the entire content of the specified header file into the current file at the location of the directive
    - <stdio.h> searches in standard directories (eg: /usr/include/)
    - `"my_header.h"` searches in the local directory first, then the system dirs
- **Phase 4** : Macro expansion (#define)
    - object_like : replaces identifiers with code fragmants (#define PI 3.14159)
    - function_like : accepts parameters and performs substitution (#define SQUARE(x) ((x) * (x)))
    - predefined_macros : Macros provided by the compiler (__FILE__, __LINE__, __DATE__)
    - stringification_# : Converts a macro parameter into a string constant
    - concatenation_## : Merges two tokens into a single token
- **Phase 5** : Conditional Compilation
    - Allows portions of code to be included or excluded based on specific conditions, often used for cross-platform compatibility or debug modes
    - directives : #ifdef / #ifndef, #if / #elif / #else, #endif
    - Include guards : A critical use case where #ifndef and #define prevent a header ile from being included multiple times in the same translation unit

- **Intermediate file generation** :
    - `gcc -E hello.c -o hello.i`
    - Contents: 
        - fully expanded source code is created in a file named hello.i
        - Line markers (eg : # 1 \"hello.c'") which tell the compiler the original line numbers for error reporting
        - Removing all #include and #define directives (they have been replaced and no longer exist)
        

- **Common Pitfalls** : 
    - Macros like SQUARE(x++) will increment  x twice because the text is substituted directly
    - Forgetting parentheses in macros can lead to logical errors 
        - `#define ADD(a, b) a+b`
        - used as ADD(1,2) * 3 = 1 + 2 * 3 = 7; instead of 9
    - SInce the preprocessor only sees text, it cannot verify if the types of macro arguments are correct
